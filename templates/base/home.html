БЛОГ

Все начинают с блога :-)

Реализовано вывод постов по времени добавления и по категориям, а также поиск по публикациям. Просматривать посты могут все. Написать статью могут только зарегестрированные пользователи. Оставлять комментарии могут только авторизированые пользователи. Есть общий подсчет постов по категориям.

Автор может вывести свои публикации, редактировать, удалять и отключать к ним комментарии, также связывать их между собой.

ИНТЕРНЕТ МАГАЗИН

Покупатель может просматривать все товары или фильтровать по категориям.

Особенность проекта в реализации корзины и заказа. Модель корзины не описана в БД, а создается виртуально и привязана к сессии конкретного покупателя, что сильно уменьшает количество запросов к БД. Добавление в корзину реализавано как отдельный сервис и не вписано как метод модели товара. В корзине реализован подсчет товара, подсчет суммы за товар одинакового типа и общая сумма заказа.

После добавления в корзину можно оформить заказ оставив данные в форме заказа. Оформленный заказ получает свой, уникальный номер, добавляется в БД и обрабатывается менеджером в ручную.

CRM (Customer Relationship Management)

Система управления взаимоотношениями с клиентами

Каждый зарегистрированный пользователь является организацией. Организация создает агентов по продаже и лидов, завязанных на конкретной организации. Лиды выводятся таблицей где есть личная инфо лида, закрепленный за ним агент и статус его обработки (новый, связались, согласился, отказал).

Организации выводятся нераспределенные лиды его организации в виде карточек с возможньостью закрепления за ними агентов по продажам. Организация может видеть статистику подсчетов обработанных лидов.

Каждый агент может пригласить своих агентов, тем самым став организацией внутри организации тем самым расширяя сеть. Агенту на почту приходит письмо с паролем, чтоб войти на сайт. Пароль генерируется автоматически и рандомно. Система для регистрации использует django сигнал. В общем этакий принцип пирамиды.

СКРАППИНГ СЕРВИС

Сервис по сбору вакансий (парсер).

Раз в сутки сервис собирает вакансии с других сайтов по ключам город-язык программирования, добавляет в БД и делает рассылку всем, зарегестрированным пользователям, кто подписался на рассылку и имеет в профиле выбраный город и язык программирования. Поиск реализован по вакансиям, добавленным в БД сервисом.

В данный момент рассылка отключена так как почта для разсылок бесплатная и годится только для тестирования.

ПУТЕШЕСТВЕННИК

Сайт по поиску маршрута по заданным параметрам (Проект из курсов на которых я познакомился с Django).

Особенность проекта в использовании теории графов для поиска и прокладки маршрутов. Также здесь есть четко поставленное ТЗ, что реально круто)))

Задание: Поиск маршрута перемещения из одной точки в другую. Задача делится на 3 части - населенные пункты, поезда, маршруты.

Населенные пункты: Реализовать добавление, редактирование, удаление населенного пункта, а также постраничный просмотр всех доступных населенных пунктов. У населенного пункта есть лишь название.

Поезда: Реализовать добавление, редактирование, удаление поезда, а также постраничный просмотр всех доступных поездов. У поезда есть уникальный код (название), начало маршрута, конец маршрута и время в пути в условных единицах. Из одной точки в другую может быть несколько поездов, но они должны отличаться по времени в пути.

Маршруты: Пользователь выбирает начальный и конечный пункт маршрута, а также указывает максимальное время в пути. Также пользователь может добавить сколь угодно промежуточных городов, через которые должен пролегать маршрут. Ему загружаются подходящие под условия маршруты. Возле каждого маршрута должна быть кнопка, позволяющая сохранить данный маршрут, задав ему имя. При поиске маршрутов, необходимо обращать внимание на направление движения поезда.

Вывод результатов: Вывод маршрутов сортируется по наименьшему времени в пути. Т.е. первым отображается маршрут, с наименьшим временем в пути. Описание маршрута должно содержать информацию о том, откуда и куда ведет этот маршрут, время в пути, а также содержать список всех поездов, которые есть в этом маршруте с указанием номера поезда, откуда\куда и времени в пути. В случае, если маршрут не найден, вывести сообщение - "Маршрута удовлетворяющего условиям поиска не существует". Если же, заданное время в пути меньше, чем минимальное время маршрута, тогда сообщение "Время в дороге больше выбранного Вами. Измените время."

Сохраненные маршруты: Должна быть отдельная страница с просмотром маршрутов. Маршрут можно только сохранить, посмотреть и удалить. Редактировать сохраненный маршрут нельзя.

Доступ к сайту: Доступ к добавлению\редактированию Поездов\Городов, а также удалению любых записей, должен быть только у зарегистрированных пользователей.

LMS (Learning Management System)

Платформа для электронного обучения.

Особенность етого проекта в универсальной модели данных которая хранит разные материалы модуля курса.

Раздел предметы, заполняет администратор с админки и создает группу инструктора. Курс делится на модули. Каждый модуль может иметь неограниченое количество материлала (материал делится на категории text, video, file, image). Редактировать и удалять курс может только владелец курса. Создавать курс может только User с правами инструктора. Просматривать список курсов может каждый. Курсы также выводятся списком или по предметам. Просматривать курс может только тот студент, который записался на курс. Студентом может быть только зарегестрированный пользователь. Розделение прав реализовано через permission_required.
